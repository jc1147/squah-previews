The whole calendar, native

Create an event. Set the date, time, and recurrence. Publish.

Events are structured content with full admin CRUD, created in the admin, published to a public events calendar, and managed from one list. No separate events plugin.

Create & edit events

A full create/edit form, title, date, start and end time, description, and an optional promo banner with CTA buttons. Publish a tasting night, a class schedule, or a venue calendar the same way you publish a blog post.

Recurrence

One-off or recurring on a Daily, Weekly, Monthly, or Yearly cadence. The recurring series is one record, not dozens of manual duplicates, manage the whole schedule by editing one event.

Categories & list management

Group events by type, workshops, tastings, live music, and run the entire calendar from one admin list, with status filters (All / Published / Draft / Trash), search, and bulk actions.

Public calendar

Every published event appears in the public events archive and gets its own single-event page, a public-facing calendar that's native, not a separate front-end plugin.

Add to calendar

Each event page has an add-to-calendar action, a Google or Outlook calendar link, or an ICS/Apple download. Visitors commit to showing up because the event lands in their own calendar with one click.

Per-event SEO

Override the SEO meta on each event so listings read the way you want them to in search and on the public page, controlled per event, in the same admin as everything else.

Questions

Frequently asked questions about Events

Can visitors add an event to their own calendar? +
Yes. Each public event page has an add-to-calendar action, it builds a Google or Outlook calendar link, or downloads an ICS/Apple calendar file. The event lands in the visitor's own calendar with one click.
Can events repeat on a schedule? +
Yes. Events can be one-off or recurring on a Daily, Weekly, Monthly, or Yearly cadence, and the whole series is managed from the single event, set a weekly class or a monthly market once and the platform generates the run.
Does each event publish a public page? +
Yes. A published event appears in the public events archive and gets its own single-event page, the native public calendar, no separate front-end plugin required.
Which plan includes Events? +
Every plan. Events is one of the 22 native modules, and every plan is the whole platform, plans differ only by how many live sites you run. Paid plans start at $39/mo (billed annually; $47 monthly), and the free Sandbox runs Events too, on unlimited staging sites.
I run more than one location. Can SGEN handle that? +
Yes. The native Location Selector module lets you manage multiple business locations from one site, alongside your events. For collecting sign-ups, pair Events with the native Forms module.
