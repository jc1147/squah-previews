Inside the module

Add, order, test, and import, all from the admin.

The redirect manager is a list-and-edit workflow built for getting a rule right before it goes live, not a blind list you hope is working.

301 or 302 per rule

Pick permanent (301) or temporary (302) on a per-rule type select. A 301 passes the old URL's ranking signals to the new one; a 302 keeps the old URL in the index for a short-term detour.

Priority ordering

Give each rule an integer priority. Higher priority evaluates first, so a specific rule can win over a broad one, and your map stays predictable as it grows.

Test URL simulator

Paste a public URL and the admin dry-runs the same matching logic the live hook uses, see exactly which rule would catch it and where it sends the visitor, without performing the redirect. Fix a wrong match before a single redirect goes live.

Hits & Last Matched

Every rule shows how many times it fired and when it last matched, so you can retire dead rules and confirm live ones. The operational visibility a paid redirect tool charges for, included.

Bulk actions & search

Draft rules inactive, test them, then activate. Select multiple rules to activate, deactivate, or delete in bulk, and search the list by partial source URL.

Import / Export (CSV or JSON)

Upload a CSV or JSON file to bring rules in for a migration, or export for backup. Import adds new rules and skips duplicate targets, import once, then edit in the UI. Your link-preservation work stays portable, not trapped in a plugin.

Questions

Questions about Redirects

Does SGEN support wildcard redirects? +
For a straightforward one-old-URL-to-one-new-URL move, use a plain 301, the reliable core of the manager. SGEN also matches with regex pattern rules rather than a separate * wildcard syntax: flip the Is-Regex toggle on a rule to match a class of URLs with a pattern, which covers the common "everything under /old-path/ → one destination" case. Test any pattern rule in the Test URL simulator before you rely on it.
Do redirects carry my link equity? +
Yes, a 301 is a permanent redirect, and search engines pass the source URL's ranking signals to the destination. That's the standard behavior of the HTTP 301 status, which is why a permanent move should use 301, and a short-term detour should use 302 (the old URL stays in the index).
Can I bulk-import my existing redirects? +
Yes, upload a CSV or JSON file in the Import / Export panel and the rules land in the list, ready to activate and test. Import adds new rules and skips duplicates (it does not update existing ones), so the workflow is import-once, then edit in the UI. Note that SGEN imports your redirect rules, not your WordPress site itself, SGEN is a rebuild platform, not a site importer.
If I rename a page's slug, does the redirect happen automatically? +
No, the redirect manager is a separate tool from the page editor, so changing a slug does not auto-create a rule. Best practice is to add a 301 from the old slug to the new one in the Redirects panel right after the change.
