The builder

Add, reorder, and nest navigation items visually.

Each menu is a tab. Build its items from your published Pages or from Custom Links, drag to reorder, and nest items into deep sub-menus. Assign any menu to Header, Footer, or Mobile, and the public site renders it there. No functions.php, no theme files.

Multiple menus, by tab

Each menu is its own tab in the builder. Build a focused header menu and a fuller footer menu separately, and assign each to the locations you want.

Pages or Custom Links

Pull published pages straight in, or add custom links with a target, same tab or new tab. Your whole nav comes from real content, not hand-typed slugs.

Drag, reorder, nest

Arrange the hierarchy visually with drag-and-drop, and nest items into deep sub-menus. The structure is something you see and move, not PHP you register.

Native mega menus

Mark an item as a Mega Menu and give it custom HTML content, for example a Shop menu that showcases categories. The plugin you'd normally license is already in the platform.

Per-item control

Set a CSS class and a link target per item, open a link in a new tab, attach a class for custom styling, without touching code. The menu does exactly what you intend.

Header, Footer, Mobile

Assign each menu to one or more of three locations. Edit the menu once and it updates everywhere that location renders, site-wide and automatically.

What you stop fighting

No PHP menu locations. No mega-menu plugin.

On WordPress, a menu lives in a theme location you register in PHP, and a real mega menu usually means a plugin. SGEN puts menus in a visual builder, separate from the theme, with mega-menu support built in, so nothing breaks when you switch themes and there is no extra plugin to license.

Questions, answered

Menus, in plain terms.

Can I create multiple menus? +
Yes. Each menu is its own tab in the builder, so you can build separate menus, for example a header menu and a different footer menu, and assign each to the locations you want.
Do menus update site-wide automatically? +
Yes. A menu is assigned to a location (Header, Footer, or Mobile), and editing that menu updates it everywhere that location renders, no per-page menu editing and no theme-file changes. Build a focused header menu and a fuller footer menu, and edit one without disturbing the other. Nothing breaks when you change themes, because menus aren't tied to the theme.
Can I build a mega menu? +
Yes. Mark a menu item as a Mega Menu and give it custom HTML content, useful for a Shop menu that showcases categories or featured items. The mega menu you'd normally license a plugin for is already in the platform, one less plugin to install, license, and watch break on update.
Can I nest sub-menus? +
Yes. Drag items to reorder them and nest them into deep sub-menus directly in the builder. The hierarchy is visual, not PHP.
Is the menu builder available on all plans? +
Yes. Menus are part of SG-Core and the Build & Design modules, included in every plan from $39/mo, and on the free Sandbox too. There is no mega-menu plugin to license.
