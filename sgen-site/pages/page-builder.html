Why a native builder

Elementor is a plugin running on WordPress, which runs on a plugin stack.

That is three failure points before you publish a single page. WordPress core updates on its own schedule. Elementor updates on top of that. Your theme, your security plugin, and your caching plugin each update on schedules of their own. One failed update in that chain can break the page you spent the afternoon building.

The named pain is Update Anxiety: plugin and core updates break the site. It is the single most common way a working page stops working. The data behind it is blunt. 91% of WordPress vulnerabilities were in plugins (Patchstack, State of WordPress Security 2026). Year after year the vast majority of WordPress vulnerabilities are not in core, they are in the plugin layer your Elementor site depends on: 96% in 2024, 91% in 2025.

The native answer: SGEN ships the builder as a native module. Native features replace the plugins that are the number-one failure source. There is no plugin stack to keep current, because there is no plugin stack.

The canvas

Drag, drop, configure, publish. Nothing underneath to update.

Open any page with Edit with SG-Builder and the visual editor opens right inside your admin. Your page appears in the center canvas, styled exactly as a visitor would see it, with a toolbar across the top and a sidebar of components and styles on the right. Click any element to select it and edit its content or settings. The builder shows your edits live as you make them. When you are ready, you click Publish.

Select, then style

Click an element. Style it live, no code.

Selecting the button on the canvas opens its settings on the right: content, typography, color, padding, and radius. Every change renders on the canvas as you make it, across desktop, tablet, and mobile widths. This is the Elementor workflow you already know, with real-product output on the other side of Publish.

Visual drag-and-drop

Rows, columns, and sections you arrange directly on the canvas. Click an element to edit its content or settings and the change renders live. The workflow you already know, with no Elementor plugin to keep current.

Sections and templates

Start from a section or a full-page template so a new page begins finished, not blank. The reusable library lives in the Templates module, so every page after the first is a head start.

Device-width previews

The builder opens to desktop width by default. Switch to tablet or mobile with the device toggle. Edits apply across all widths unless you set a per-device override, so you catch the mobile break in the editor.

Live media

Swap a photo or add an image and the builder pulls from your Media Library. Upload once, then drag it into place on the canvas.

Auto-saved drafts

The builder auto-saves your edits as a draft as you work, separate from publishing. An interrupted session is preserved, and nothing goes live until you click Publish.

What ships to the browser

Server-side rendering delivers finished content instantly, to users and to Google.

SG Builder is where you design. Server-side rendering is what you publish. The server renders the complete page before it leaves the data center, so the visitor and Google's crawler both receive finished content. There is no client-side render step they have to wait on. It is not a caching plugin and not a configuration option. It is how the platform works.

A builder is only as fast as what it outputs. You get drag-and-drop convenience without the JavaScript-render penalty. Speed and SEO are built into the output, not bolted on with another optimization plugin.

Questions

Frequently asked questions about SG Builder

Can I use my own fonts and brand colors? +
Yes. Use any Google Font or upload your own through the Custom Fonts module, self-hosted, with no third-party font plugin. Site-wide typography, color, and spacing are set in the Theme Editor so your brand stays consistent across every page, not re-set page by page.
Will my pages break when SGEN updates? +
Plugin and core updates are the number-one way WordPress pages break. 91% of WordPress vulnerabilities were in plugins (Patchstack, State of WordPress Security 2026). SGEN's answer is Immutable Stability: native features replace the plugins that are the failure source, so there is no plugin stack to break on update.
Can I start from a template? +
Yes. The Templates module gives you reusable page and section templates, so a new page starts finished rather than blank. When you open the builder, you can pull a section or full-page template straight into the canvas.
Does my draft get lost if my session drops mid-edit? +
No. The builder auto-saves your edits as a draft as you work, separate from publishing. If a session is interrupted, your draft is preserved and you can pick up where you left off the next time you open that page. Your edits go live only when you click Publish.
Can I preview how a page looks on mobile before publishing? +
Yes. The builder opens to desktop width by default. Use the device toggle in the top toolbar to switch to tablet or mobile preview. Your edits apply across all device widths unless you set a per-device override.
Do I need WordPress, or a separate host, to use SG Builder? +
No. SG Builder is a native module on the SGEN platform. There is no WordPress underneath and no separate hosting invoice. Hosting, SSL, CDN, WAF, security patching, and the multi-site dashboard are in the Foundation Pack, free on every plan including the $0 Sandbox.
