Customer story

Fernwood Digital: 23 client sites, one stack, one login.

A 12-person web and marketing studio in Portland rebuilt its entire 23-site client book on SGEN over a single quarter. The plugin towers came down, the maintenance treadmill stopped, and the team got its design practice back. The numbers below are illustrative of the story this fictional example brand tells.

The challenge

A design studio that had quietly become a plugin-maintenance company.

Over nine years Fernwood accumulated 23 active client sites, every one on WordPress, and every one assembled the same way: a page-builder plugin for layout, plus a forms plugin, an SEO plugin, a popup plugin, a caching plugin, a backup plugin, a cookie-consent plugin, a call-tracking plugin, and whatever else a client had needed that quarter.

The problem was never any single site. The problem was 23 of them, each on its own update schedule. A plugin would push an update on a Tuesday and silently break a checkout form on one site; a core update would white-screen another. Because each site carried a slightly different stack, there was no single fix. There were 23 fixes, found one panicked client email at a time.

Plugin tax, multiplied by 23

Premium licenses for forms, SEO, popups, call tracking, and backups stacked up on every site, renewals landing in different months on different cards. Nobody owned the total, and it climbed with every new client the studio won.

No portfolio-level view

There was no single dashboard. Checking on a client meant logging into that client's WordPress admin, hunting for credentials, and hoping the last freelancer had not changed something.

Update roulette

Every update was a gamble, so the team started delaying them out of fear, trading the risk of a broken site for the risk of an unpatched security hole. Per industry context, Patchstack reports that 91% of WordPress vulnerabilities sit in plugins.

What they built on SGEN

One all-inclusive platform. No plugin tax, no suite tax.

Fernwood rebuilt the client book on SGEN as one stack, not a base product plus add-ons, and not a per-site pile of plugin licenses. The whole platform came in the box: all 22 native modules across 7 categories. The rebuild ran roughly two sites a week and finished in a quarter. It was a rebuild, not an import: each site re-laid on SGEN's visual builder from house templates, then a redirect map carried search equity over.

One screen, the whole client book

The multi-site dashboard ships free in the Foundation Pack. All 23 client sites land in one fleet view: status, traffic, forms, and backups, from a single login instead of 23 separate WordPress admins. This is the feature that ended the question of which login belonged to which client.

What they consolidated onto

Native modules replaced the per-site plugin stack, one for one

Templates. A small set of house section and page templates, built once, so a new client site starts finished, not blank.
Backups. On-demand and daily backups with retention, per site, from the same dashboard, in place of the per-site backup plugin.
Users and roles. Designers, account managers, and clients each get the right scope natively, with no third-party roles plugin to license and patch on every site.
Login as client. When a client emails about a broken form, the team drops straight into that client's site as the client sees it, no credential hunt, no shared passwords.
Foundation Pack. Hosting, free auto-renewing SSL, a global CDN, a WAF, and automatic security patching on every site, so the patch anxiety that had Fernwood delaying updates simply went away.
