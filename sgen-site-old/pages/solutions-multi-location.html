The multi-location reality, and the native fix

Every location you add on WordPress is another site to keep alive.

A separate install per location

Each location is its own WordPress site: its own login, its own hosting, its own updates. Ten locations is ten things that can go down independently, on ten different Tuesdays.

The SGEN fix

One site with the native Location Selector, plus a multi-site dashboard to manage everything from a single view. No install-per-location sprawl.

A plugin stack multiplied by N

Whatever plugin stack runs one location runs at every location, and every license, every renewal, every security patch is multiplied by your location count.

The SGEN fix

Native modules, configured once. Forms, SEO, backups, and the rest are part of the platform, so there is no plugin license to stack per location and your maintenance burden stops multiplying.

Brand and information drift

Hours change at one location and not the others. A promo goes live in three places and stays stale in two. With separate installs, consistency is a manual chore nobody owns.

The SGEN fix

Per-location content where it should differ (address, phone, hours) and shared site structure everywhere else, so the brand stays consistent without copying changes by hand.

91% of WordPress vulnerabilities were in plugins (Patchstack, State of WordPress Security 2026). Multiplying a plugin stack multiplies the most common breach vector.

Location Selector

Every location, with its own details, managed in one place.

Each location is a structured record with its own name, full street address, phone number, email, hours of operation for every day of the week, and gallery images. A public location selector lets a visitor choose their location, and the site remembers that choice for the visit, so people see the address, hours, and contact details that actually apply to them.

Manage, search, and filter every location from one admin list. Brand and information stop drifting: hours change at one location without going stale at the others, and consistency stops being a manual chore nobody owns.

Questions

What multi-location operators ask before switching.

Is multi-location an add-on, or is it included in the plan? +
It is included. The Location Selector is a native module that ships in every plan, so there is nothing to add on and nothing priced separately. Plans start at $39/mo (billed annually; $47 monthly) for one live site, and the whole platform, multi-location included, is in every tier.
Do I need a separate install for each location? +
No, that is the model we replace. It is one site with the native Location Selector, plus a free multi-site dashboard if you are running several sites. One platform, not one install per address.
Can each location have its own hours, address, and contact details? +
Yes. Every location is its own record with name, full address, phone, email, gallery, and hours of operation for each day of the week, and the public selector shows visitors the location that is actually theirs.
How does pricing work if I run several locations on separate sites? +
Pricing is flat per site-count tier, not per location and not per module. Every plan includes the whole platform, multi-location included, and you pick the plan by how many live sites you run: Indie (1 site, $39/$47), Studio (3 sites, $97/$117), Scale (10 sites, $297/$333), or Enterprise (unlimited, $497/$577). If your locations share one site with the Location Selector, one plan covers them. If each location is its own live site, pick the tier that covers your site count. A free unlimited Sandbox is available for staging.
Can I move our existing per-location WordPress sites over? +
Moving to SGEN is a rebuild, not an import. You consolidate onto one platform rather than dragging N plugin stacks with you. There are two paths: a white-glove rebuild where we rebuild it for you, or a DIY rebuild on the SG Builder with templates. Talk to sales for rebuild pricing.
