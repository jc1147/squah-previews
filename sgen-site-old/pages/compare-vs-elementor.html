What it means to run "on WordPress"

Elementor is a plugin on WordPress. That sentence is the argument.

When you build on Elementor, you are working inside a stack, and every layer is a thing you maintain or a thing that can fail.

WordPress core. Updates on its own schedule, conflicts with its own plugins.
WordPress hosting. A separate vendor with its own bill.
Elementor. Its own update cycle, on top of the WordPress update cycle.
Your theme and other plugins. Each one a failure surface.

Elementor One bundles more of this together. But the underlying architecture is the same: Elementor is a plugin running on WordPress core, and it always has been. SGEN ships the builder as a native module. There is no WordPress underneath, no WordPress hosting vendor, no WordPress core updates to manage. One platform, one update event.

Common questions

Objections, handled.

Does SGEN have the same design flexibility as Elementor? +
SG Builder is a visual drag-and-drop builder with column-and-row layout, inline editing, and design controls at the site, section and element level. It is built as a native module on the SGEN platform, not as a plugin on WordPress. The best way to evaluate fit for your workflow is to build a page on the free Sandbox and see it in action.
Will my Elementor templates transfer? +
No. Moving to SGEN is a rebuild, not an import. Elementor's template format is proprietary to the WordPress and Elementor stack, and SGEN does not import Elementor templates. The rebuild path uses SGEN's native template library as a starting point. See the migration guide for the full rebuild walkthrough.
Do I still need hosting if I move to SGEN? +
No. Hosting is included on every plan, including the free Sandbox, so you do not pay a separate hosting invoice on SGEN. The infrastructure an Elementor-on-WordPress setup charges for separately (hosting, SSL, CDN, WAF, patching, the multi-site dashboard) is the floor here, from $39/mo.
What plugins does SGEN replace? +
In a typical Elementor and WordPress stack, SGEN replaces the page builder (SG Builder, native), security plugins like Wordfence and Sucuri (native WAF and patching), form plugins like WPForms or Gravity Forms (Forms module), popup plugins like OptinMonster (Popups module), SEO plugins like Yoast (On-Page Optimization module), and attribution tools like CallRail (Attribution module). Multi-touch attribution and phone-tap logging are native, so you can see which campaign drove a lead inside the same platform you build in, with no third-party invoice stacked on the build.
Is Elementor One the same as the old Elementor plugin setup? +
Elementor One is a more bundled offer than the older plugin-only setup. But the underlying architecture is unchanged: Elementor still runs on WordPress core. Your page builder is still a plugin on WordPress, a platform that carries its own update cycle, its own security surface and its own hosting dependency. SGEN replaces the entire stack with one native platform.
