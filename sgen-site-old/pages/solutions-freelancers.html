The WordPress tax

Three problems that eat every client engagement.

A typical freelance client site is WordPress, plus Elementor Pro, plus a forms plugin, plus a caching plugin, plus a security plugin. That is four licenses, four update schedules, and one weekend of panic every time an update breaks the live site the night before a client meeting. Each problem below is a known, solvable thing on SGEN, not a plugin to install.

Four plugins, four invoices, one fragile stack

Every client site inherits the same overhead: a builder license, a forms license, a caching plugin, a security plugin. Each is a separate subscription and a separate update risk. SGEN ships the builder, forms, performance controls, and security natively. Nothing to stack, nothing to license per site.

The midnight plugin breakage, you know the one

Core and plugin updates ripple through the stack in ways nobody can predict. A client emails at 7 a.m. the day before launch: "the site is down." SGEN runs on Immutable Stability. Native features replace the plugins that are the primary failure source, so when there is no plugin stack to break, there is nothing to break.

A slow site costs the client traffic, and costs you the referral

WordPress assembles pages in real time with PHP and leans on a caching plugin to hide it. The result is slow-to-first-byte pages that hurt Google rank and bounce real visitors. SGEN renders pages server-side: finished content delivered instantly to users and to Google, with no caching plugin to install or tune.

Visual, drag-and-drop. Precise enough to match any design.

SG Builder is a visual, drag-and-drop builder with the precision of Elementor and the output of a real product. Common questions

What freelancers ask before switching

Can a client log in to manage their own site after I hand it off? +
Yes. SGEN has role-based access control built in, no permissions plugin required. You create the client a profile and set their role to Editor: they can edit their pages and publish posts, while layout, settings, and your other clients' sites stay locked to you. The roles available are Administrator, Editor, and Customer, and the Users module is native to SG-Core on every site.
How fast can I ship a new client site? +
The workflow is: create a Sandbox, start from a template, customize in SG Builder, connect the domain, go live. Templates start the build from a finished layout, and the builder is visual, so you skip the steps that eat the most time on WordPress: plugin configuration, child-theming, and chasing plugin conflicts. We do not publish a "ship in X hours" number, because any specific figure would be a claim we cannot verify. What is verifiable: the platform removes the slowest steps of a WordPress build.
What if the client wants to take the site elsewhere later? +
SGEN sites are built on the SGEN platform: the builder output, hosting, and modules are part of the platform, so a SGEN site is not a WordPress export you drop onto any host. If a client moves off SGEN, that is a rebuild on the destination, the same way moving onto SGEN is a rebuild rather than an import. Be straight with clients about this up front: SGEN trades portability for a maintenance-free, plugin-free platform. For most freelance clients, a small-business site they want to stop worrying about, that trade is the point.
Does SGEN handle hosting for the client site? +
Yes. Hosting is part of the Foundation Pack, included free on every plan, including the $0 Sandbox, along with SSL, CDN, WAF, and security patching. You do not buy separate hosting per client; the plan covers the infrastructure.
Can I migrate my client's existing WordPress site to SGEN? +
A migration to SGEN is a rebuild, not an import: there is no WordPress-content importer. Two paths: a white-glove rebuild, where SGEN's team rebuilds the site (contact us for white-glove pricing), or a DIY rebuild, where you rebuild it on SGEN with the builder and templates. A rebuild produces a clean, plugin-free site rather than carrying WordPress technical debt across. If the client has hundreds of posts, plan for a content-migration scope: it is a project, not a one-click transfer.
Is SGEN secure enough for a client's live business site? +
The Foundation Pack includes a WAF and security patching on every plan, and the native feature model removes the biggest source of WordPress risk. As Patchstack's State of WordPress Security 2026 report found, 91% of WordPress vulnerabilities were in plugins. A site with no plugin stack has no plugin surface to exploit. SOC 2 and ISO are in progress; we report status, not certifications we do not hold, and there are no certification badges, just a compliance-grade posture. For specialized compliance, talk to sales.
