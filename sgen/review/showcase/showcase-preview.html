The Challenge

A design studio that had quietly become a plugin-maintenance company.

Fernwood Digital is a 12-person studio that designs and maintains websites for local businesses — dental groups, law firms, a regional grocery chain, a handful of restaurants. Over nine years the agency had accumulated 23 active client sites, every one on WordPress, and every one assembled the same way: Elementor for layout, plus a forms plugin, an SEO plugin, a popup plugin, a caching plugin, a backup plugin, a cookie-consent plugin, and a call-tracking plugin.

The problem was never any single site. The problem was 23 of them, each on its own update schedule. A plugin would push an update on a Tuesday and silently break a checkout form on one site; a WordPress core update would white-screen another. Because each site had a slightly different stack, there was no single fix — there were 23 fixes, found one panicked client email at a time. Premium licences for forms, SEO, popups, call tracking, and backups stacked up on every site, renewing in different months on different cards, with nobody owning the total.

The Solution

One all-inclusive platform — no plugin tax, and no suite tax.

Fernwood rebuilt the client book on SGEN as one platform — not a base product plus à-la-carte add-ons, and not a per-site pile of plugin licences. The whole stack came in the box. The migration was a rebuild, not an import: Fernwood re-laid each site on SGEN's visual builder from its house templates, then ran a redirect map so search equity carried over. Roughly two sites a week, finished in a quarter.

The agency leaned on the Agency / multi-site capability set — every feature below is a real, native part of the platform, no add-on to license.

Multi-site dashboard
All 23 client sites in one fleet dashboard — status, traffic, forms, backups — from a single login. Free on every plan via Foundation Pack.
Login-as-client
Impersonation drops the team straight into a client's site as the client sees it — no credential hunt, no shared passwords.
Users & Roles (RBAC)
Designers, account managers, and clients each get the right scope of access natively — no third-party roles plugin to patch.
Templates
A small house template kit so a new client site starts finished, not blank.
Backups & restore
On-demand and daily backups with retention, per site, from the same dashboard.
Attribution
First-party, native — each client can see which campaign actually drove their leads. No CallRail-style per-line tax.
Foundation Pack
Hosting, free auto-renewing SSL, global CDN, a WAF, and automatic security patching ship free on every site — so the patch anxiety simply went away.
The Results

The maintenance treadmill stopped — and attribution became a retention tool.

  • ✓23 client sites consolidated onto one all-inclusive stack and one fleet dashboard.
  • ✓An average of 9 paid plugins per site eliminated — forms, SEO, popups, caching, backups, cookie consent, call tracking, a roles plugin, and a security plugin — all replaced by native features.
  • ✓Cross-portfolio maintenance dropped from ~14 hours/week to ~2 — the freed time went back into billable design work and a new retainer tier.
  • ✓Zero "the-update-broke-the-site" emergencies in 8 months, versus roughly two a month before.
  • ✓New-client time-to-launch fell from ~3 weeks to ~5 days, working from the house templates library.
  • ✓Attribution renewed 6 wobbling retainers — clients can now see the marketing working, in a leads-by-source view.
