Components & form states

Every reusable component, shown live in every relevant state — not described, not screenshotted-elsewhere, actually rendered here with real working CSS. Buttons carry their full state machine (default · hover · active · focus-ring · disabled · loading). Form controls show default · focus · filled · error · success · disabled. Feedback, disclosure, data and nav components each appear in all their states, on the SGEN chrome: charcoal #1a1a1a, red #d51522, one 8 px button radius, the ink-tinted shadow family, motion gated behind prefers-reduced-motion.

Interactive states work — hover / tab-focus / click any real control. States marked PINNED are force-applied via a modifier class so one screenshot captures them all.

You're billed monthly on the plan you choose. Upgrades are prorated instantly; downgrades take effect at the next cycle. This row is the open state — note the accent title and the rotated chevron.

Header / nav states — see the dedicated gallery

The full header / nav state machinery — the sticky is-stuck ↔ at-top boolean, the transparent→solid flip, the hamburger's aria-expanded true/false, and the off-canvas drawer's nav-open / aria-hidden states (with scrim, scroll-lock, Esc and submenu accordion) — is demonstrated live and in motion across four production header blocks in sections-headers.html. Header states are scroll-driven, so a static specimen here can't prove them; the gallery scopes each header in its own scroll frame so every state flips on demand. The 44 px floor above is the same target the gallery's hamburger and drawer close buttons meet.

Carousel controls — prev/next (rest · hover · focus) + dots (inactive vs active)

Real <button> controls, 44 px circular. Rest = hairline; hover = accent border + text; focus-visible = the --ring. Hidden on mobile (swipe replaces them).

The active dot fills with accent and stretches to a pill; inactive dots are neutral discs. Built by JS per page-count, synced to the track's scrollLeft, rebuilt on resize.
