B

The mobile-nav drawer — the #1 gap

A real, working off-canvas hamburger → drawer. Click the hamburger inside the phone and the drawer slides in via translateX, a full scrim dims behind it, scroll locks, the submenu accordion expands, and it closes on the close button, scrim-tap, link-tap, or Esc. Embedded in a phone-frame so it works inline at any viewport. 44px tap targets throughout; driven by aria-expanded / aria-controls.

⚠ This is the single biggest mobile pattern — and the most under-built

A naive build "reflows" the desktop menu into a vertical list. The correct build is a separate position:fixed panel parked off-screen (transform:translateX(100%)), a real scrim with its own click-to-close, a hard scroll-lock, and a submenu toggle that is a separate control from the link (tap the label to navigate, tap the + to expand). The demo here parks the panel inside the phone frame instead of the page viewport so it stays self-contained.

D

Key patterns, live

Four high-leverage moves you can verify directly: sticky-header shrink-on-scroll, the clamp type ladder, 44px tap targets (good vs bad), and full-bleed band vs capped reading column.

D1Sticky header — shrink on scrolllive
● Brand mark 90px → 60px
▼ scroll this panel — the header shrinks 84→56, gains a shadow + solid bg

The bar starts tall and transparent-feeling. On scroll it adds .stuck — height drops, the logo shrinks, a shadow appears, and the background goes solid. The modern upgrade swaps BeTheme's flat opacity:.97 for real backdrop-filter glass.

Scroll position drives a single class toggle via a passive scroll listener — no layout thrash.

Keep scrolling. The sticky bar stays pinned at the reduced height while content flows beneath it.

The same trick covers "transparent at top, solid when stuck" with one rule: .site-header:not(.is-sticky){background:transparent}.

On mobile the header is forced solid + static — never carry a transparent overlay into phone widths.

More content to give you room to scroll and watch the transition settle.

And a final paragraph so the scroll has somewhere to go.

Keys on ≥768. 90px static → 60px sticky · logo 60→50 · solid bg + 0 2px 5px shadow · z-index 30→701. Transition the change.

D4Full-bleed band vs capped reading columnlive

The band bleeds — the words don't.

The dark band spans the full width of its container, but the reading column inside it caps and centers. On a 1920 monitor the band fills the viewport while the text stays ~520px wide and readable. Body text never runs 1700px wide.

section = 100vw · reading column = capped + centered

"Full width" almost always means full-width backgrounds/sections with a capped reading column. Above the cap there is no stretch query — content pins, gutters grow, bands fill. Bump the token for a roomier ultrawide; never unbounded.

E

The per-breakpoint correctness checklist

Every SGEN site must pass this at all six widths. Fail-closed: an unproven item is a failing item.

⚠ The gate — render & read at 390 AND 768, fail-closed

A green desktop screenshot proves none of these. Stacking, drawer-open, overflow, tap-target size and type legibility are only verifiable by rendering at 390 AND 768 and reading the capture. An unproven item is a failing item — default to broken until shown green.

∀Universal — 390 → 1920
  • No horizontal scroll — html{overflow-x:hidden} present; no full-bleed spawns a 1–17px scrollbar.
  • Container caps + centers for the tier (1220 / 940 / 708 / 700+33px); reading column never exceeds ~1220 even at 1920.
  • min-width:960 floor released to 0 below 1240.
  • Type legible — heading ladder applied; body ≥13px, lh ≥1.5; inputs ≥16px; longest hero line fits.
  • 44px tap targets on every interactive control.
  • Motion gated by prefers-reduced-motion; background-attachment:fixed killed ≤959 / iOS.
  • Retina via srcset/image-set(), not a density @media.
390Mobile portrait
  • Every multi-column row single-column, full-width, 20px gaps (except compact tiles).
  • Hamburger visible (44px); desktop nav hidden; drawer opens, scrim dims, scroll locks; Esc/scrim/close dismiss; submenu accordion works.
  • Hero stacked; 100vh released to height:auto; CTA full-width / ≥44px.
  • Header solid + static; mobile logo; right-cluster collapsed (cart keeps icon, drops $ total).
  • Footer fully stacked; legal row centered; fixed footer un-pinned; back-to-top in 20px gutter.
  • 33px gutters; section padding ~0.6–0.7×; eyebrows don't wrap.
768Tablet portrait
  • Container = 728/708 (visibly narrower than landscape).
  • Grids at an explicit 2-up — NOT desktop-count, NOT free-fallen to 1-up.
  • Side-by-side internals stacked + centered (icon-box, counter, image|copy).
  • Nav is the drawer (or desktop bar only with a chosen 1024 swap — no half-collapsed bar).
  • Footer still multi-column (tablet-one-third) — does NOT stack until 767.
  • At exactly 768, columns are NOT auto-stacked (1px above the 767 line).
1024Tablet landscape / small laptop
  • Container = 960/940; line-length readable (no edge-to-edge text).
  • Grids at the laptop step (6/5 → 4-up); internals stay horizontal.
  • Nav: hamburger (BeTheme default) OR desktop bar if 1024 is your swap line — decided.
  • Touch safety: parallax→scroll, :hover fallbacks, tel: links, 44px targets.
1200Laptop
  • Same tier as 1024 — container still 960/940; no separate 1200 grid.
  • Full desktop nav back if your collapse line is 1024; still hamburger if 1240 — confirm it matches.
  • Cards/grid at full or near-full desktop count; no cramped nav bar.
1440Desktop
  • Full 1240/1220 container, centered; base (largest) type; full horizontal hover nav.
  • Sticky+shrink works (90→60, logo 60→50, solid bg + shadow); transparent→solid flip if used.
  • All desktop-library depth/motion/type intact — this is where the ceiling is judged.
1920+Full-bleed / max-width cap
  • Content pinned at ~1220 + centered with wide gutters — body text NOT 1700px wide.
  • Full-bleed bands fill the viewport (100vw/cover); reading column capped.
  • No breakage above 1440; ultrawide cover gated ≥1921; reads as a fixed-width centered document with edge-to-edge bands.
