Page 15 — the data-viz kit · GAP #7

Numbers, drawn in the brand.

The system had count-up stats and progress bars but no chart primitives. This closes the gap: a small CSS/SVG data-viz set — bar chart, line / sparkline, donut & ring gauge, KPI metric card. Every colour is derived from the one accent plus the four semantic tones; every animation is reduced-motion-aware; every chart is responsive and self-contained — system fonts, zero dependencies, CSS-and-SVG visuals, no images.

How it works

Each bar's height is just height: calc(var(--v) * 100%) on a flex/grid column — no JS measures anything. The only script toggles one .in class when the chart scrolls into view, which flips scaleY(0) → scaleY(1).

02Line & sparkline

Inline SVG — a polyline over a gradient area fill. The line draws itself on with stroke-dashoffset; the area fades up behind it; the end-point dot pops last. The same engine shrinks to a 108×30 inline sparkline that lives inside a list row or a metric card. Trend colour is semantic — green up, red down.

A sparkline carries no axis, so colour does the talking: green when up is good, red when down, accent when neutral. The 30px height drops straight into a table cell or a KPI card with no layout shift.

05Colour tokens — derived, not invented

The whole kit reads from one token block. The series ramp starts at the accent and adds de-conflicted hues; the semantic tones (shared with page 07) carry status. Reskin the accent and every chart re-tints with it.

One accent, the whole kit

Bars, the line stroke, the ring sweep and the gauge gradient all anchor to --accent. Status — a KPI delta, a sparkline trend, a quota ring — switches to the semantic tone. Nothing here is pure #000 or #fff; the grid is tinted ink so the data always wins the contrast.

What holds it together

Four primitives, one discipline.

The same rules that govern the rest of the styleguide govern its charts — depth from light not borders, motion gated behind reduced-motion, a single accent over a committed neutral.

01

Derived colour

Every hue traces to the accent or a semantic tone. No bespoke chart palette to drift out of brand.

02

Reduced-motion-aware

Grow-ins, draw-ons and sweeps fire only when motion is allowed; otherwise the final state is painted instantly.

03

CSS & SVG only

No charting library, no canvas, no images. A polyline, a circle, a calc()-driven height — that is the whole engine.

04

Responsive by default

viewBox SVG scales fluidly; grids collapse to one column; gauges shrink. Verified 390 → 1440.
