Build sequence · For the engineering team

A wave plan that respects dependencies.

A proposed order of build that climbs the ladder while front-loading the shared foundations every later suite depends on. Sequencing is a recommendation — yours to set.

◇

The dependency that gates everything: a unified client/account data model

Every agency-facing capability — operations, billing, marketing, the client portal — assumes one thing the per-site CMS doesn't have yet: a client/account entity that spans sites (the agency → client → sites → tasks → invoices graph). Build it first and the rest of ACMS clicks together. Skip it and every feature reinvents it.

WAVE 0
Harden the base + agency spine

Finish the CMS hub — document it, make multi-site seamless, close the restore-from-URL SSRF + CSRF gaps — and formalize the multi-site / provisioning APIs. Then stand up the spine the whole ACMS product hangs on: the unified client/account data model, cross-site RBAC, and agency-owner login-as.

The operate-my-agency MVP: client management + support desk + project management (tickets become per-client tasks, team boards) + client billing/invoicing on the internal payment rail. This is what lets staff run the whole client roster end-to-end on SGEN.

WAVE 2

The authoring engine — AI text + image generation + chatbot, and the Guided Build Workflow — built on the SG-Builder + templates + media repo we already have. Also Track B's demo-led sales engine; dogfood it from day one.

DEPENDS ON · SG-Builder (live) · templates (live) · media repo (live) · AI services horizontal
WAVE 3
ACMMS · Marketing engine

The recurring-revenue engine — the strongest reason an agency renews. GBP management + AI review/content posting + citations (integrate Local Falcon); SEO AI-management + gap analysis; plus the automated service workflows and lead engine, fired from site events.

DEPENDS ON · Locations CMS (live) · SEO module (live) · Analytics + Site Vitals (live) · AI services · Wave-0 client model · Local Falcon API
WAVE 4
Finish ACMS + AERP

The monetization + retention polish: e-sign (integrate), reseller economics, and the branded client portal pulling reports + task status from every suite above — all running on the internal payment gateway (AERP), the financial backbone the platform bills through.
