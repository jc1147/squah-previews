A native content-type builder

Define any structured content type your site needs.

The Custom Objects module owns five jobs in one admin: define the type, attach its fields, organize it, publish it, and route it. Register the type, attach the fields, and give it a taxonomy from one screen, point-and-click, with nothing to install.

Content-type builder

Register a type with a name, a slug, and its publishing behavior, no code. Each type gets its own Pages browser, its own URL structure and index, plus its own single, archive, and loop-item templates, a permalink-structure control, items-per-page and items-per-row layout, SEO defaults, and header and footer script injection.

Custom Fields, native

Build a group of fields and attach it to any content type, a Page, a Blog post, an Event, or one of your Custom Objects, and the fields render on that type's edit form. 15 field types, from text and number to image picker, repeater, and nested group. Structured data lives in the same dashboard, nothing to install.

Full object CRUD

Every type gets a six-state index: All Objects, Published, Draft, Private, Password Protected, and Trash, with live counts, inline search, bulk publish, draft, or trash, and per-row edit, trash and restore. The workflow you already know from posts, applied to your own content types.

Categories per type

Give any type its own taxonomy with a two-card Categories manager: an inline create-and-update form on the left, a paginated searchable table on the right. Categorize listings by neighborhood or recipes by cuisine and build filtered archives from the category your visitor picked.

Portfolio as a preset

A portfolio is a Custom Objects preset, not a separate engine: define a Portfolio Projects type, attach the fields you care about, client, category, date, hero image, outcome, and publish each piece as an object. It inherits the same fields, categories, and routing your other types use.

One consistent admin

Define the type, attach its fields, set its categories, and publish its objects from the same admin where you manage pages and posts. Object pages open in the SG Builder visual editor, the same builder your pages use, so there is one controller and one UI to learn.

The fields, native

A native field builder. Fifteen field types. Native.

SGEN's Custom Fields module builds a group of fields and binds it to any of four content types. Define field rows with a label, a name, and a type, set the group's locations and status, and attach a listing's price and bedrooms or a recipe's prep time and servings, with nothing to license.

The palette goes well past plain text and number. Repeatable fields add Item #1, Item #2 rows on the edit form for galleries and feature lists; nested Group fields model a structured sub-record inside one field; the Image and Attachment types are native Media-library pickers that return the file with inline upload and preview; and the HTML type is a full rich-text WYSIWYG editor for long-form body copy, formatted in place, no shortcodes to memorize. These five premium types carry the depth a generic add-a-field toggle hides.

Questions

Frequently asked questions about Custom Objects

Is Custom Objects the same as WordPress custom post types? +
It fills the same role, defining structured content types beyond pages and posts, but the implementation is native. You register the type point-and-click and wire no schema by hand. You define the type in the Custom Objects admin and SGEN generates the full admin CRUD, the templates, the permalink pattern, and the per-type categories.
Do I need a developer to create a content type? +
No. The module is a point-and-click admin: give the type a name, a slug, and its publishing settings, and SGEN generates the rest, the single, archive and loop templates, the items-per-page and items-per-row layout, and the per-type SEO. Fields are added through the Custom Fields admin, also without code.
What is Custom Fields, is it a separate product? +
It is a native companion module, not a separate purchase: a field-group builder with 15 field types (text, number, select, image picker, repeater, nested group, and more) that you attach to any content type including your Custom Objects. The fields render on that type's edit form, and each field carries a front-end shortcode. Native, in the box.
Can I organize a content type with categories? +
Yes. Every type gets its own Categories taxonomy with a create-and-update form and a searchable category table, so you can categorize listings by neighborhood or recipes by cuisine and build filtered archives, all native.
Which plan includes Custom Objects? +
Every plan. Custom Objects and Custom Fields are two of the 23 native modules, and every plan is the whole platform; plans differ only by how many live sites you run. Paid plans start at $39/mo (billed annually; $47 monthly), and the free Sandbox runs the module on unlimited staging sites.
Is there anything Custom Objects does NOT do? +
It is a content-type and content-modeling engine, not an e-commerce catalog or a headless API product. Bulk status changes (publish, draft, trash) and per-row delete are supported, and categories and the per-type templates are configured in the admin. Object pages open in the SG Builder visual editor, the same builder your pages use, not a separately-licensed tool. For product catalogs with inventory and checkout, use the e-commerce module instead.
