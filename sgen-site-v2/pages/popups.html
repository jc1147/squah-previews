More than a newsletter widget

A real content tool for on-site modals, not a single email toggle.

A newsletter widget gives you one rigid template and a subscribe field. SGEN Popups is a managed content type: a full editor or the visual page builder for the body, a Configuration rail for how it shows, a hashcode to fire it from anywhere, and a draft-to-publish list with author and created metadata. Native in your dashboard, with no per-popup license.

Two ways to author

Write the body in the full WYSIWYG text editor, the verified-stable default, with bold, lists, alignment, links, images, tables, and a source-code view, or switch the Editor source to the SG Builder visual path, in beta, to design the popup with the same drag-and-drop builder as your pages.

A hashcode for every popup

Each popup gets a copy-paste #sgp_<ID> hashcode. Drop it as the href on any link or button anywhere on the site to open that popup on click, alongside the optional autoload.

Configuration rail

Set the title tag (H1 through H6 or paragraph), an explicit width in pixels, extra CSS classes, and toggles for the close button, overlay-close, stacking, popup title, and inline versus AJAX content.

Per-session autoload

Turn on autoload and set a per-session frequency: 0 shows it on every page, or cap it to the first N times a visitor sees it in a session. It is configured right on the popup itself, native to the module.

Draft-to-publish workflow

Status tabs for All, Published, Draft, and Trash with live counts, search by title or date, bulk publish, draft, trash, and a Trash-only delete-permanently. Soft-delete keeps a popup recoverable.

Shortcodes, and Forms inside

The popup body accepts [shortcodes], so a form from the Forms module renders right inside the popup and its submissions land in the same Forms inbox. Two native modules, working together.

Questions

Frequently asked questions about Popups

How do I trigger a popup from a link? +
Every popup has a hashcode shown in the list and on its edit screen, in the form #sgp_<ID>. Paste it as the href on any link or button anywhere on the site, and clicking that element opens the popup. You can also turn on autoload so the popup opens on page load.
Can I design a popup with the page builder? +
Yes. The Editor source on the right rail offers a Text Editor, any of your appearance templates, or SG Builder. Choose SG Builder and an "Edit with SG Builder" button opens the same drag-and-drop visual builder you use for your pages, so the popup body is designed, not just typed. The Text Editor is the verified-stable default; SG Builder authoring is the standout option and is in beta, where the canvas opens empty on edit, so check the popup after you build it.
Can a popup contain a form from the Forms module? +
Yes. The popup body accepts [shortcodes], so paste a form's shortcode into the Content area and that form renders inside the popup. Its submissions land in the same Forms submissions inbox, so the popup and the form are two native modules working together.
What can I configure on a popup? +
The Configuration rail sets the popup title tag (H1 through H6 or paragraph), an explicit width in pixels, extra CSS classes, and toggles to disable the close button, disable closing on the overlay, disable stacking multiple popups, hide the popup title, and switch between AJAX-fetched and inline content. Autoload adds a per-session frequency cap.
Does autoload show the popup on every visit? +
You decide. Enable autoload and set a frequency: 0 shows it on every page, or set a number to show it only the first N times in a session. Without autoload, the popup opens only when a visitor clicks a link bound to its hashcode.
Can I unpublish or remove a popup? +
Yes. Set its status to Draft to take it off the site, or move it to Trash as a soft-delete that you can restore later. Delete Permanently is available on the Trash tab. Note that restoring a trashed popup returns it as a Draft, so re-publish it when you want it live again.
When does a published popup go live? +
Immediately. Publishing a popup makes it appear on the public site with no cache or build step, rendered as a centered modal over your page content. Moving it to Draft or Trash removes it from the live site just as quickly.
