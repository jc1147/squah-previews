Why a snapshot beats hoping

Roll back to any point in time, on demand.

The Backups panel turns your whole site into a portable, named, point-in-time archive you can return to. One click captures everything; the list keeps each snapshot dated and ready. No backup plugin to license, no FTP, no host-specific add-on to schedule and keep current.

One click, the whole site

Create a Backup bundles a full SQL dump of your database plus your entire uploads directory into one .sgen file. Nothing to select or assemble: pages, posts, products, settings, and media all travel together.

A row you can return to

Every snapshot lands in the list with its filename, size, and exact creation date, for example backup-2026-04-23-1402.sgen, 142 MB, Apr 23 2026. You always know what state you are restoring.

Rename snapshots in place

Click a filename to turn it into an editable field, type a meaningful name like post-product-launch-v2, and press Enter to save. The server sanitizes it to a safe, unique .sgen name.

Find any snapshot fast

The Search box filters by filename substring, case-insensitive, and the list paginates 10 rows per page, newest first, sorted descending by creation time. A long history stays easy to scan.

Clear out old snapshots in bulk

Check several rows, choose Delete under Action For Selected, and hit Apply. The panel confirms Deleted N backup(s). Deletion is permanent, so treat it as housekeeping, not an undo.

Native, nothing to install

Backups are part of the platform, one of 23 native modules, not a plugin you add, license, and keep patched. The panel updates with the platform and adds nothing extra to maintain.

Questions

Frequently asked questions about Backups

What does a backup include? +
Create a Backup captures a full snapshot: a complete SQL dump of your database plus your entire uploads folder, bundled into one .sgen archive, so pages, posts, products, settings, and media all travel together. Full-site snapshots in the panel run roughly 124 to 142 MB.
How do I organize my backups? +
Rename any snapshot by clicking its filename and typing a meaningful name, then press Enter to save. Search the list by filename substring, and clear out old snapshots in bulk with Action For Selected, Delete, Apply. The list shows 10 rows per page, newest first.
How does restore work? +
Restore returns your site to a captured point in time from the snapshot you choose. Because a restore replaces the live database, uploads, and admin users in place, and there is no in-app rollback once it runs, SGEN treats recovery as a platform-assisted operation: you raise it with your platform team rather than firing an irreversible self-serve button. There is no dry-run or preview, so it is deliberate, not casual.
Are backups automatic or off-site? +
No, and this is the honest limit: every backup is a manual snapshot you trigger with + Create a Backup, stored on the same site storage. There is no schedule, no automatic cadence, and no off-site copy in this module. Treat it as an on-demand safety net you fire before risky changes, not a set-and-forget rotation. Deleting a snapshot is permanent, with no trash and no undo.
Can I download a backup as a zip to my computer? +
No. There is no self-serve download-a-zip button; snapshots live in the Backups panel as portable .sgen archives you restore from there. We state this plainly so nobody expects a download action the product does not render.
Is backup included on every plan? +
Yes. Backups are part of the platform, and every plan is the whole platform, all 23 native modules. Plans differ only by how many live sites you run. Paid plans start at $39/mo (billed annually; $47 monthly), and the free Sandbox lets you build and snapshot before you pay.
