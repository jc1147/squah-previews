Who we are

We built the WordPress alternative we wanted to use.

WordPress runs a large share of the web, and that reach is exactly where the trouble starts. The moment a site needs to do real work, it becomes a stack of plugins: one for forms, one for SEO, one for backups, one for security, one for attribution. Every plugin is a separate license, a separate update, and a separate way for the site to break. We did not set out to add another plugin to that pile. We set out to make the pile unnecessary.

The why

Why we built it.

Every operator we talked to told a version of the same story. A WordPress site that worked, until it slowly did not. A plugin update broke the checkout. The next update rewrote the SEO titles. A security review flagged plugin vulnerabilities buried in a stack nobody had fully mapped, some of them in plugins the site had stopped using but never removed. The site was never broken by one big thing. It was broken by the architecture that made adding one more plugin the answer to every requirement.

That architecture has a measurable cost. Every plugin is third-party code on its own release schedule, its own security history, and its own way of breaking when the platform underneath it moves. The site was never undone by one bad plugin. It was undone by depending on dozens of them, each one a license to renew, a dependency to track, and an update that could take the page down on a morning you did not choose.

So we asked a different question: what would a modern publishing and commerce platform look like if every core capability shipped natively, the page builder, SEO, forms, ecommerce, attribution, backups, security, popups, events, redirects, 23 modules across 7 categories, with no third-party licenses to stack? Server-side rendering from the first request, so finished content reaches the visitor and Google instantly, not a caching plugin bolted on after the fact. First-party multi-touch attribution and call tracking built in, where a WordPress site would reach for a separate subscription. A single dashboard for every site you run, where WordPress would hand you a separate admin login for each.

What it means: you build a full publishing-and-commerce site without assembling a stack of plugins, each with its own license and update. Every capability ships when the product ships and patches when the platform patches, with no dependency breaking on its own schedule. That is what SGEN is: a faster WordPress alternative, built for operators who are done debugging plugins.

How we think about web software

Four things we believe.

No icons, no slogans. These are the calls we make every time a feature could be native or could be one more dependency you maintain.

A native feature beats a plugin every time.

A capability built into the platform is more reliable, has fewer moving parts, and is one less invoice than the plugin that does the same job. We build everything native first, and when something genuinely has to be external, we integrate it deliberately, not by plugin roulette. A native feature ships when the product ships and patches when the platform patches, with no third-party dependency to track or breach. Stop depending on dozens of plugins and the failure pattern they create is simply not yours to carry.

Speed is an architecture decision, not a hosting upsell.

Server-side rendering is not an add-on you buy at a higher tier. Smart-Loading Logic is not a premium caching plugin. Performance is something you design in once and ship to everyone, so we made it the default, free on every plan including the $0 Sandbox, rather than a line item you budget for and maintain.

Your data belongs in your platform.

Attribution, analytics, consent logs, call tracking: these have been farmed out to third-party vendors and monthly subscriptions for too long, each one holding a slice of data you should own. We ship them natively, including multi-touch attribution and call tracking with no separate call-tracking tax, because the operator running the site should hold the numbers that explain it.

Compliance is baseline infrastructure, not a feature you buy when you get big.

Audit-ready logging, GDPR consent management, a WAF, and ongoing security patching are not rewards for scaling. They are the floor, shipped free in the Foundation Pack on every plan including the $0 Sandbox. You start with a compliance-grade posture in place from day one, even on the free tier.
