Native module · Custom Objects

Any content type. Native.

SGEN's Custom Objects module models any content type your site needs, property listings, recipes, team bios, portfolio pieces, as native structured data with its own admin, slug, URL structure, and index. Define the type point-and-click and SGEN generates the single, archive, and loop templates, the permalink pattern, the per-type categories, and the full draft-to-trash admin.

Attach the data with the native Custom Fields builder and its 15 field types, with no CPT UI plugin to register the type and no ACF license to attach the fields. Each type runs the same lifecycle you already know, a six-state index with live counts, bulk actions, search, and per-row edit, trash and restore, and its content items open in the SG Builder visual editor.

A native content-type builder

Define any structured content type your site needs.

The Custom Objects module owns five jobs in one admin: define the type, attach its fields, organize it, publish it, and route it. No code to register a content type, no license to attach the fields, no plugin to give it a taxonomy.

Content-type builder

Register a type with a name, a slug, and its publishing behavior, no code. Each type gets its own Pages browser, its own URL structure and index, plus its own single, archive, and loop-item templates, a permalink-structure control, items-per-page and items-per-row layout, SEO defaults, and header and footer script injection.

Custom Fields, native

Build a group of fields and attach it to any content type, a Page, a Blog post, an Event, or one of your Custom Objects, and the fields render on that type's edit form. 15 field types, from text and number to image picker, repeater, and nested group. This is the ACF role, native, nothing to license.

Full object CRUD

Every type gets a six-state index: All Objects, Published, Draft, Private, Password Protected, and Trash, with live counts, inline search, bulk publish, draft, or trash, and per-row edit, trash and restore. The workflow you already know from posts, applied to your own content types.

Categories per type

Give any type its own taxonomy with a two-card Categories manager: an inline create-and-update form on the left, a paginated searchable table on the right. Categorize listings by neighborhood or recipes by cuisine and build filtered archives, no taxonomy plugin.

Portfolio as a preset

A portfolio is a Custom Objects preset, not a separate engine: define a Portfolio Projects type, attach the fields you care about, client, category, date, hero image, outcome, and publish each piece as an object. It inherits the same fields, categories, and routing your other types use.

One consistent admin

Define the type, attach its fields, set its categories, and publish its objects from the same admin where you manage pages and posts. Object pages open in the SG Builder visual editor, the same builder your pages use, so there is one controller and one UI to learn.

The stack it replaces

One module. The whole content-type stack, in the box.

SGEN's Custom Objects ships the type registration, the field attachment, and the per-type taxonomy as one native module, on every plan. A typical WordPress custom content type is a small renewing stack instead: one plugin to register the post type, a second to attach the field groups, a third for custom taxonomies, three update cadences and three renewal dates that have to stay in sync. Questions

Frequently asked questions about Custom Objects

Is Custom Objects the same as WordPress custom post types? +
It fills the same role, defining structured content types beyond pages and posts, but the implementation is native. There is no CPT UI plugin to register the type and no post-meta schema to wire by hand. You define the type in the Custom Objects admin and SGEN generates the full admin CRUD, the templates, the permalink pattern, and the per-type categories.
Do I need a developer to create a content type? +
No. The module is a point-and-click admin: give the type a name, a slug, and its publishing settings, and SGEN generates the rest, the single, archive and loop templates, the items-per-page and items-per-row layout, and the per-type SEO. Fields are added through the Custom Fields admin, also without code.
What is Custom Fields, is it a separate product? +
It is a native companion module, not a separate purchase: a field-group builder with 15 field types (text, number, select, image picker, repeater, nested group, and more) that you attach to any content type including your Custom Objects. The fields render on that type's edit form, and each field carries a front-end shortcode. Native, no ACF.
Can I organize a content type with categories? +
Yes. Every type gets its own Categories taxonomy with a create-and-update form and a searchable category table, so you can categorize listings by neighborhood or recipes by cuisine and build filtered archives, no taxonomy plugin.
Which plan includes Custom Objects? +
Every plan. Custom Objects and Custom Fields are two of the 23 native modules, and every plan is the whole platform; plans differ only by how many live sites you run. Paid plans start at $39/mo (billed annually; $47 monthly), and the free Sandbox runs the module on unlimited staging sites.
Is there anything Custom Objects does NOT do? +
It is a content-type and content-modeling engine, not an e-commerce catalog or a headless API product. Bulk status changes (publish, draft, trash) and per-row delete are supported, and categories and the per-type templates are configured in the admin. Object pages open in the SG Builder visual editor, the same builder your pages use, not a separately-licensed tool. For product catalogs with inventory and checkout, use the e-commerce module instead.
