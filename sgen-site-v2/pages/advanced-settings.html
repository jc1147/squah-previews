Native module · Site Settings

Site configuration. One panel.

The SGEN Site Settings panel runs your whole site identity from one form: Site Title, Site Description, Email, Site Logo and logo hyperlink, Favicon, a Membership toggle, Timezone with a live local-time readout, plus Set Homepage and Set 404 Page. One Save Changes button persists the Identity, Business Information, and Age Verification cards together, roughly 28 to 29 fields, and the change is live on the next pageview with no cache warmup and no redeploy.

Three tabs sit on the same rail: General for identity, Email for SMTP and sender, Social Media for profiles and icons. On WordPress the SMTP mailer, the social metadata, and the snippet settings each arrive as a separate plugin with its own settings page and its own update cycle.

The configuration surface

The settings that scatter into plugins, native, in one panel.

The same Save Changes button persists Identity, Business Information, and Age Verification together, and identity values fan out site-wide. One panel covers the configuration a WordPress site usually reassembles from a mailer plugin, a metadata plugin, and a snippets tool.

General & identity

Site Title, Site Description, Email, Site Logo, Favicon, Timezone, Set Homepage, and Set 404 Page, plus a Business Information card (phone, address, opening hours, Google CID) that emits LocalBusiness JSON-LD and the public footer contact block.

Email and SMTP

From Email, From Name, and a bring-your-own SMTP block, so forms and notifications send from your own provider. Run a test send before you launch instead of finding out in production.

Social Media

Per-platform Name and URL plus an Icon chosen from a 33-icon Feather-style catalog. Add Social Link appends an item, Save Changes persists, the profile links are managed in one place.

Age Verification

A site-wide gate in popup or full-page landing mode, with custom Yes and No button text, a No-button redirect, and a cookie expiry in days. Self-declaration, configured in the same form.

Import / Export

Export Pages, Blog posts, and Events to one JSON file, then import back through a JSON or CSV dropzone, with a Basic tier that preserves draft or publish status and an Advanced tier with visual field-mapping.

Search & Replace

A database-wide find and replace for a domain move or a brand rename. Dry-run it first to see the tables scanned and rows matched, and a backup is taken before any write, so you preview the blast radius before you commit.

Cross-field fallbacks

Empty From Email falls back to the site Email, empty From Name to the Site Title, and an empty page SEO title to the Site Title plus tagline. The defaults are real, so a half-filled form still sends and renders.

Age Verification

A native age gate, popup or full page.

A site-wide age gate for regulated industries (alcohol, cannabis, 18+) in either popup or full-page landing mode, with custom Yes button text, custom No button text, a No-button redirect URL, a configurable disclosure Content editor, an optional gate Logo, and an Expiration days value that controls how long a Yes click is remembered in a cookie. The public render reads "Are you 21 or older?" with a confirm button and a "No, take me back" link.

This is a self-declaration prompt. It is not ID verification and not legal or regulatory age proof. Use it as a gate, not as compliance.

Database-wide edits, safely

Find and replace across the site. Dry-run first, backup before any write.

The Search & Replace tool, a sibling of Post Migration on the Tools rail, edits content across the database in one pass, the kind of job you reach for after a domain move or a brand rename. Type what to find and what to replace, pick exactly which tables to touch, and run a dry-run first to see the tables scanned and rows matched before anything is written. A backup is taken before the replace commits. The pass is URL-aware, so it rewrites links even when they are slash-escaped inside JSON content, and customer, order, user, session, and form-submission tables are kept out of the picker by design.

Questions

Frequently asked questions about Advanced Settings

Does this include SMTP, and can I test it before launch? +
Yes. Set your From name and address, connect a standard SMTP provider such as Google Workspace, Microsoft 365, SendGrid, Mailgun, or Postmark, choose NONE, SSL, or TLS encryption on a chosen port, and run a Send Email test from the Email Testing card to confirm delivery before you go live. Treat it as a test send that proves the route, not a guarantee of inbox placement.
Can I export my content out, with no lock-in? +
Yes. The Import / Export Pages tool exports Pages, Blog posts, and Events to one JSON file. Select all or pick per content type, and the record counts on screen match what downloads. You can import back through a JSON or CSV dropzone, with a Basic tier that preserves draft or publish status and an Advanced tier with a visual field-mapping step that groups targets into Standard fields and Post custom fields.
Can I find and replace across my whole site safely? +
Yes, with guardrails. The Search & Replace tool runs a database-wide find and replace, but you pick exactly which tables to touch, and a dry-run reports the tables scanned and rows matched before anything is written. A backup is taken before the replace commits, so you can preview the blast radius and keep a rollback point. The pass is URL-aware, so it catches links that are slash-escaped inside JSON content, and customer, order, user, session, and form-submission tables stay out of the picker by design. It is built around dry-run and backup, not a one-click case-insensitive replace, so run a real write only after the preview looks right.
Does the age gate make my site legally compliant? +
No. The age gate is a native, configurable self-declaration prompt, popup or full-page landing, with custom buttons, a No-button redirect, and a cookie expiry. It is not ID verification and not regulatory or legal age proof. Use it as a gate, not as compliance.
Can I add custom head or body code here? +
No. Advanced Settings does not inject snippets. Head and body code injection lives in the separate Custom Codes module, and Custom CSS is its own admin utility. This panel stays scoped to identity, email, social, the age gate, and content export.
Is there an API key or integrations connector in this panel? +
No. The settings panel covers identity, email, social, the age gate, content export, and Search & Replace. The older Integrations and SEO Config routes are retired, the SEO Manager now lives in its own area and Google Integrations under Tools, so there is no API-key surface inside this module.
Is Advanced Settings on every plan? +
Yes. It is one of the 23 native modules, and every plan is the whole platform, plans differ only by how many live sites you run. Paid plans start at $39/mo, and the free Sandbox runs the full settings panel, SMTP, social, the age gate, export, and Search & Replace, on unlimited staging sites.
