Native module · Custom CSS

Write it once. It lives on a real screen.

Custom CSS is a site-wide override manager with four parts that work together: a real editor, an organized list, a cascade control, and live site-wide injection. Four jobs, one native screen.

A real editor, not a paste box

Write your CSS in a syntax-highlighted, line-numbered code editor with lint markers and a per-snippet Title. The editor is CodeMirror with a CSS mode, so selectors, properties, and values color as you type.

A managed snippet list

Every snippet lives in a populated, searchable list with Active, Inactive, and Trash status, live status counts, bulk actions, and per-snippet Author, Created, and Modified metadata. The list-management you already know from posts, applied to your CSS.

A priority cascade

Give each snippet a Priority from 1 to 20 so multiple stylesheets cascade in a controlled order, not one undifferentiated box. A base override at priority 1, a campaign tweak at priority 10, both live, both in order.

Goes live site-wide

Active snippets inject into every public page's head, theme-independent, so the same overrides apply across your whole site and survive a theme switch. Change a snippet's status and it goes live across every page.

Frequently asked questions about Custom CSS

Is this the same as the WordPress Customizer's Additional CSS box? +
It fills the same need, site-wide custom CSS, but it is a native screen in the platform, not a textbox tied to your active theme. It does not vanish when you switch themes and it is always present, including on modern block themes. You get a real code editor, a snippet list, a status workflow, and a priority cascade instead of a single box.
Will my CSS survive a theme switch? +
Yes. Snippets are stored at the platform level, theme-independent, not in the active theme, so changing themes does not remove them. That platform-level storage is why your overrides keep applying through every theme switch.
Do I need a plugin to add custom CSS? +
No. Custom CSS is one of the 23 native modules, on every plan. There is nothing to find, vet, install, or license, you open the Custom CSS screen under Appearance and start writing.
Where does my custom CSS apply? +
Site-wide. Active snippets inject into every public page's head, so overrides apply across your whole site. For styling a single page or block, use SG Builder block styles instead, Custom CSS is the site-wide override manager.
How do priority and status work? +
Each snippet has a Status, Active, Inactive, or Trash, and a Priority from 1 to 20. Active snippets emit, inactive ones stay saved but do not emit, and trashed ones can be brought back. Priority controls the cascade order when multiple snippets are active.
Is there anything Custom CSS does NOT do? +
It is a site-wide CSS override manager, not a per-page styler and not a full theme editor. For single-page or single-block styling, use SG Builder block styles, and to edit your theme's own template files, use the Theme Editor. Custom CSS is for site-wide style overrides that load in the head.
