One content engine, native

A content team on WordPress is a stack of plugins. Each one is an update vector.

Set up a content-team site on WordPress and the bill comes due in plugins: a page-builder plugin to edit a page, an image-optimization plugin for WebP, a permissions plugin for roles, a mega-menu plugin for navigation, a theme-builder plugin for global design. Every one updates on its own schedule, and every update is a chance for the site to break.

SGEN ships those capabilities as native parts of one platform. Page building, image optimization, roles, navigation, and global design are built in, governed by a single update cycle. You maintain your whole content stack as one product, instead of reconciling five plugins that each update, and each break, on their own timeline. Fewer moving parts means fewer things to patch and fewer ways for a routine update to take your site down.

Questions, answered

The core CMS, in plain terms

Does the core CMS replace Gutenberg and a page-builder plugin? +
Yes. SGEN ships a native page and blog engine, and SG Builder is a native visual editor. There is no WordPress core, no Gutenberg, and no page-builder plugin underneath. You build pages and publish posts inside the platform itself.
Is the core CMS available on the free Sandbox? +
Yes. The core CMS is part of every plan, including the free Sandbox, and the Foundation Pack is included on the Sandbox too. The whole platform ships on every plan; the only thing a Sandbox site cannot do is go public-live on a custom domain, since it runs on a staging URL. Upgrade to a paid plan for that.
How many native modules come with the platform? +
23 native modules across 7 categories, on top of the six core content areas and the dashboard operating layer. None of them is a third-party plugin.
Do I still need plugins for media optimization, roles, or menus? +
No. Media optimization (WebP and AVIF), role-based access control, and the visual menu builder are all native CMS capabilities, managed in the same dashboard, with nothing to install and nothing to keep current.
Is the Theme Editor a live capability today? +
Yes. The Theme Editor capability is live in the admin: typography, global colors, layout, and your header and footer are all editable site-wide. It is part of the core CMS, managed in the same dashboard as your pages and posts.
