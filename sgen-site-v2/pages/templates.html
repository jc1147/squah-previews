Edit the source block in one place and every page that references it updates. Or assign a template as your site's Header, Footer, or Mobile Menu and it becomes the chrome across the whole site. Full CRUD, a trash and restore lifecycle, bulk actions, JSON import and export, and an optional SG Builder visual canvas, included on every plan.

The problem with copy-paste blocks

A footer you pasted onto forty pages is forty footers to fix.

Most site tools treat reusable content as a snippet you copy in. The moment a phone number, a promo line, or a legal link changes, you are hunting through every page that holds a copy. Worse, the ones you miss drift out of sync, and there is no record of where a block is even used.

SGEN Templates makes the block itself the single source. The page holds a reference, not a copy. Change the source once and every reference follows. Trash one by mistake and restore it. Move your whole catalog to another install as JSON. It is a library with a lifecycle, not a pile of pasted HTML.

What the module does

One library. Three jobs done natively.

Build, govern, and deploy reusable content from one screen, native in your dashboard with nothing to install and no per-template fee.

Build it once

Author a block or a full page layout in the rich-text editor, or flip the content source to SG Builder and lay it out on the visual canvas. Set a title, choose Published or Draft, and save. One template, one source of truth.

Govern the lifecycle

A real Draft and Publish status, one-click Duplicate that clones to a Draft with metadata preserved, AJAX Trash and Restore with a live Trash tab and counts, and Delete Permanently as the irreversible step. Bulk-select to move many at once.

Deploy two ways

Drop the [template id="N"] shortcode inline on any page, or assign the template as your site Header, Footer, or Mobile Menu through Appearance. The same block can be a reusable inline snippet and the chrome that wraps your whole site.

The manage screen

Title, status, content source, and a shortcode you can copy.

Creating or editing a template is one form. Name it, pick Published or Draft, choose the rich-text editor or SG Builder as the content source, and write. On an existing template you get a read-only shortcode chip to copy, the created and last-modified dates, and a clear warning if this template is the active Footer, so editing its layout will not surprise you.

Manage the catalog in bulk

Select many, move them all in one action.

A growing library needs more than one-at-a-time edits. Check the templates you want, pick an action from Action for Selected, and Apply moves every selected row at once: Move to Publish, Move to Draft, or Move to Trash. On the Trash tab the action set switches to Delete Permanently, so a seasonal cleanup is a single pass, not forty clicks.

Pair it with the status pills and the search box and the list is a real working catalog: filter to Draft, select the batch you finished, and publish them together. The same screen handles one template or a hundred.

Questions

Frequently asked questions about Templates

What is the shortcode and how do I use it? +
Every saved template gets a shortcode like [template id="3"]. Paste it on any page, post, product, or event to embed that template's content. Because the page references the template instead of copying it, editing the source template updates every place the shortcode appears.
Can a template become my site Header or Footer? +
Yes. Through Appearance you can assign any published template as the site Header, Footer, or Mobile Menu, so it renders as chrome across the whole site. The edit form shows a warning when a template is the active Footer so you know your changes affect every page.
Can I build a template visually instead of in a text editor? +
Yes. Set the content source to SG Builder and the template opens into the full visual canvas, the same editor behind the Page Builder module. You commit changes with the Publish Changes button. Templates set to the rich-text editor use the standard content editor instead.
How do I move templates to another SGEN site? +
Use Import / Export. Export one template or all published templates as a JSON file, then import that file on the other install. The export carries each template's status and metadata, so the imported templates arrive with their settings intact. Note that import does not de-duplicate by title, so re-importing the same file creates additional copies.
What happens when I trash and restore a template? +
Trash moves a template to the Trash tab, where it is held until you Restore it or Delete Permanently. Restore brings it back as a Draft, so a template that was Published will need to be re-published after you restore it. Delete Permanently is the irreversible step and only applies to trashed items.
Can I duplicate a template? +
Yes. Duplicate clones the template to a new Draft named "<original> (Draft)" with its metadata preserved, so you can branch a variation without touching the original. You can also bulk-select multiple templates and move them to Publish, Draft, or Trash in one action.
