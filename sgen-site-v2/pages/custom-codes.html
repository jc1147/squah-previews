What it really is

A managed snippet library, not a paste box.

Custom Codes is a real script, pixel, and tracking manager. Every snippet is a titled row you can search, filter, activate, deactivate, trash, and restore, with author and timestamp attribution. That is a full library with a Trash, not the single header-and-footer textarea the phrase usually means.

Titled rows, not a textarea

Each snippet is a named library row, not one big box. Give it a clear title, Google Analytics 4 or Cookie Banner, and find it again by name. The list shows Title, Status, Placement, Priority, Author, and Created on every row.

Status filter tabs

Four tabs with live count badges, All Custom Codes, Active, Inactive, and Trash, so you see at a glance what is live, what is parked, and what is in the bin. The counts are real database state, not labels.

Search and bulk actions

Search the library by title, then use Action For Selected and Apply to set rows active, inactive, or trash in one pass. Single-row Trash and Restore give you a recoverable step for one snippet at a time.

Author and timestamp

Every row carries an author email and a Created time, so on a shared site you can see who added the analytics tag and when. Attribution is built into the list, not bolted on.

Three load placements

Set head, body start, or body end per snippet, so an analytics loader fires early and a heavy widget loads late. The sibling Custom CSS module injects stylesheet rules the same way, from the same kind of managed library.

Priority ordering

When two snippets share a placement, a priority number from 1 to 20 sets the order, lower fires first. That is deterministic control over load order, the kind of detail a generic add-a-code line hides.

Native module · Custom Codes · the editor

The editor is a line-numbered, syntax-highlighted code editor, not a bare textarea, paired with a Tips panel that tells you exactly where each kind of snippet belongs. It directly answers the question that brought you here, where do I put my pixel.

Frequently asked questions about Custom Codes

Where do I paste my Google Analytics tag? +
In the head or at body end, and the Tips panel guides you: head for the analytics loader that must run before render, body end for tags you don't want to block paint. Paste the snippet, pick the placement, set it Active, and it injects across your whole site.
Can I add a Meta or Facebook pixel? +
Yes. Create a snippet, paste the pixel code, and place it at body start or head per the pixel's own instructions. It is a managed library row you can deactivate or trash later without touching any theme file.
Will my code survive a theme change? +
Yes. Snippets live in the Custom Codes module, not in your theme files, so switching or editing your theme leaves every snippet in place and still injecting.
Can I target a single page only? +
No. Snippets are global, they inject site-wide across your whole site, not per-page or per-URL. Set that expectation up front: Custom Codes is the site-wide head and body injector, not a per-page rule engine.
Do I need a plugin? +
No. Custom Codes is one of the 23 native modules, built into every plan. The single most-installed WordPress escape hatch (Insert Headers and Footers / WPCode on 2M+ sites) is a plugin you license and update; here it is a built-in screen with nothing to install.
What happens when I delete a snippet? +
It goes to Trash after a Delete Confirmation step, not straight to gone. Restore brings it back as Inactive, so it stays off your live site until you re-activate it, a safety net, not a surprise re-injection.
