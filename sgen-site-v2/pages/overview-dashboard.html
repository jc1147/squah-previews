What WordPress operators never had

WordPress scatters the operating layer across a plugin stack. SGEN puts it in one panel.

On a typical WordPress build, the controls in that dashboard live in separate plugin admin menus, each with its own settings page, its own update cycle, its own license. The dashboard is whatever you can keep open in tabs. SGEN's operating layer is native: no plugin licenses, no update roulette. Every Tuesday-update scramble across a dozen plugin dashboards becomes a non-event, because there is no operating-layer plugin to break.

The operating layer you run here simply does not carry that surface, because the capabilities are native, not plugins. There is no stack of third-party operating-layer plugins to add, license, patch, or reconcile, so the maintenance burden and the attack surface that come with that stack are not part of how SGEN works.

Questions

Questions about the dashboard

Is the dashboard an extra plugin I have to install? +
No. The operating layer, Site Manager, Stage & Live, Analytics, Advanced Settings, and Backups, is native to the platform. There is nothing to install, license, or update.
What does the dashboard actually show me at a glance? +
Your homepage's Site Vitals score (PageSpeed, with Core Web Vitals), leads from the last 30 days by source, open SEO issues, and a recent-activity feed of content changes. Site Vitals auto-refreshes on a schedule rather than in real time, so use dedicated uptime monitoring if you need instant outage alerts.
Does the multi-site dashboard cost extra? +
No. The multi-site dashboard is part of the free Foundation Pack, included on every plan, including the $0 Sandbox.
Which plan do I need for the dashboard? +
The operating-layer areas appear from the free Sandbox upward. Specific per-area plan details, for example which plan includes scheduled backups, are confirmed on the pricing page.
