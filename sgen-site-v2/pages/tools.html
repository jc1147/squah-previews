Native module · Tools

Run your whole site from one operations toolbox.

SGEN groups seven operations tools under a single Tools node and reads them back as a three-column screen: the dark sidebar with Tools expanded and red-active, an in-content card-list that repeats all seven with one-line descriptors, and the active tool's own panel. The bare Tools route 302-redirects to the first sub-tool, so you always land on real work, not a settings index.

The seven are Google Integrations, Search and Replace, Post Migration, Maintenance Mode, Site Protection, Optimization, and Misc Tools, with a sibling Migration node beneath them for full-site backups. The work of running a site lives on a few admin screens instead of a stack of plugins, each with its own update cycle.

Operator-grade, native

The work of running a site, on a handful of admin screens.

The Tools cluster owns the jobs a site needs after it launches: tune how pages are served, move content in and out, swap a domain across the database, hold the front door during a cutover, gate a staging build, and snapshot the whole site before a risky change. Performance tuning, search and replace, and full-site backups are native modules in one engine, sharing one login and one update cycle.

Performance you can read

The Optimization tool carries 6 settings cards and around 14 controls for caching, minification, deferral, lazy-loading, and LCP media preloading. The Dashboard Site Vitals card reads the result back in four metrics.

Content in and out

Export your whole library to one JSON file with exact counts, or import via a Basic JSON path that keeps draft and publish status, or an Advanced path with a visual field-mapping step for CSV or JSON.

Database find and replace

Swap a domain or a brand name across the whole database from a per-table picker, with a dry-run preview first and a timestamped SQL backup written before any real change.

A go-live pair

Maintenance Mode swaps the public site for a holding page you edit in a real HTML editor. Site Protection gates a staging build behind a single access key while you build.

A safety net, in one click

The Backups tool captures the database and the uploads directory into one .sgen archive, listed on a clean table you can rename and restore from. Take a point-in-time snapshot before anything risky.

Platform health

Misc Tools reclaims database space by clearing post revisions, and a Database Checker verifies that every required table is present. Keep the platform tidy from one place, no maintenance contract.

Search & Replace · database find/replace

Swap a domain or rebrand across the whole database, with a dry-run first.

The Search & Replace tool runs a find-and-replace across your database from three cards: what to replace, which tables to touch, and the run options. Scope the change with a per-table picker that shows each table's size, preview it with the dry-run toggle before any write, and let the tool save a timestamped SQL backup the moment you run it for real.

Migration · Backups

Snapshot your entire site, content, settings, and media, in one click.

The Backups tool captures the whole site, the database plus the uploads directory, into one .sgen archive, listed on a clean table. Take a point-in-time snapshot before any risky change, rename a snapshot inline so you remember why you took it, and keep a safety net built into the platform rather than a separate backup product.

Questions

Frequently asked questions about Tools

What are the seven tools, and where do they live? +
All seven sit under the Tools node: Google Integrations (Search Console & APIs), Search & Replace (database find & replace), Post Migration (Import / Export Post Types), Maintenance Mode (temporary offline), Site Protection (password protect), Optimization (performance settings), and Misc Tools (utilities). Full-site Backups live on the sibling Migration node beside Tools. It is an operations toolbox, not a settings index, and the bare Tools route sends you to the first sub-tool.
How does the Optimization tool improve performance, and how do I see the result? +
The 6 settings cards tune how pages are served: caching with a lifespan, HTML, CSS, and JS minification with per-URL exclude lists, CSS and JavaScript deferral, lazy-loading with an above-the-fold skip list, and LCP media preloading. The Dashboard Site Vitals card reads the result back as First Content Paint, Avg. Load Speed, Performance Score, and Speed Index. Site Vitals is a stored snapshot from the last scan, not a live monitor, so treat the numbers as a captured read-out, not a guaranteed score.
Can I move my content in and out, and in what format? +
Export the whole library (Pages and Blog) to one JSON file with exact counts, shown as a running total before you download. Import via a Basic JSON path that keeps draft and publish status on the way in, or an Advanced path that takes CSV or JSON with a visual field-mapping step that auto-suggests source column to target field matches, grouped into Standard fields and Post custom fields. Export is JSON-only despite the CSV label on the format field, and import writes immediately with no dry-run, so back up first.
Is the database find-and-replace safe to run on a live site? +
Yes, run it with the built-in dry-run preview and the automatic timestamped SQL backup that is written before any real change, plus the per-table picker so you scope exactly what gets touched. What it does NOT do safely: the case-insensitive option is not safe for partial matches, since it can change the casing of a whole field, so leave it off for anything but an exact, case-matched swap. This is database text replacement, not a migration importer.
What do Backups do, and what do they NOT do? +
Backups create a one-click full-site .sgen snapshot, the database plus uploads, that you can list, rename, and restore as a deliberate action. What they do NOT do: there are no scheduled, automatic, or off-site backups, every snapshot is manual and kept on the same disk; there is no self-serve download of an archive; and restore is not a one-click reversible button, since it replaces the live site, so save a fresh snapshot first and treat recovery as a deliberate, platform-assisted operation. Delete is permanent, with no trash.
Which plan includes the Tools cluster? +
All of it is native on every plan, part of the 23 native modules. Plans differ only by live-site count: Indie is $39 a month annual, $47 monthly (1 site); Studio $97 / $117 (3 sites); Scale $297 / $333 (10 sites); Enterprise $497 / $577 (unlimited). A free unlimited Sandbox on staging URLs runs the full toolbox before you pay anything.
