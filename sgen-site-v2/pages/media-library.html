The asset hub

Browse, filter, sort, and search every file in seconds.

The library opens to a card grid or a list view and holds every image, video, document, and downloadable file on your site. The same store feeds pages, posts, products, and the blog, so you upload once and pull the asset anywhere instead of hunting through a folder of duplicates.

Upload once, reuse everywhere

Files here are available to pages, posts, products, and the blog from the same library. Upload an asset once and place it anywhere, with no re-uploading duplicates and no version drift.

Filter by 6 types

The type filter holds All, Images, Videos, PDFs, Docs, and Text, and auto-submits the moment you change it. Six named filters, so the number is concrete, not a vague "any file".

Sort and search

Sort by Most Recent, Name A-Z, or File Size, and search the moment you start typing. Search matches the stored file name and the original name you uploaded, so a file is never lost behind a rename.

Grid or list, with where-used

List view adds an ON-USE column showing where a file is referenced, beside Preview, File Name, Uploaded, Size, and Action, so you can see what a delete would break before you click it.

Copy public link

Grab any file's public URL in one click, ready to paste into an email, a campaign, or anywhere off-site. No download-and-rehost step to share a single asset.

Four actions per file

Preview, Edit, Copy Link, and Delete sit on every card and every row. The action you need is one click away, in the same view where you found the file.

Optional, on your terms

Turn on AI alt-text suggestions when you want them.

Media Settings keeps one switch, Enable AI Media Captions, that ships off. Turn it on and new image uploads get a suggested caption for accessibility and search, a first draft you review and refine, not auto-text you have to trust. When no AI caption is produced, the alt text falls back to a cleaned-up version of the filename, so a missing result is never a blank.

It is provider-configured and may use a billable AI call, so you decide when it runs. Editorial control stays with you, every suggestion is editable in the same Media Information form.

Questions, answered

Media Library, in plain terms

Does the library convert images automatically? +
Yes, to WebP on upload through the Image Optimization control, which offers Keep Original, Convert to WebP, or Compress Images. A 2 MB photo lands near 198 KB with no visible quality loss, and the conversion is built into the upload step, nothing extra to install or keep current.
Is Media Library on the free Sandbox? +
Yes. It is part of the core CMS and included on every plan, including the $0 Sandbox. The library and WebP optimization are included everywhere. Serving files from a Google Cloud bucket is an optional, config-gated cloud-storage add-on, not the default.
Can it write alt text or captions for me? +
Optionally. Turn on Enable AI Media Captions in Media Settings and new uploads get a suggested caption you review and refine. It ships off, you keep editorial control, and a missing result falls back to a cleaned-up filename. A head start you approve, not auto-text.
What file types and sizes can I upload? +
JPG, PNG, GIF, WebP, MP4, MOV, AVI, PDF, DOC, DOCX, XLS, and XLSX, up to 10 MB each, with images over 2048px resized automatically. The uploader validates real file content and rejects disguised files, so an SVG or a renamed executable is blocked, not stored.
What does Media Library not do yet? +
There is no self-serve Import/Export screen and no multi-select bulk delete in the grid today, so you delete files one at a time behind a confirmation. Deletes are permanent with no trash or undo, and removing a file does not auto-fix links to it in existing pages or posts. Check the list view's ON-USE column before you delete.
Is everything built in, or do I add tools on top? +
Everything is built in. Upload, organization, search, WebP conversion, alt text, copy-link, and optional AI captions are all native to the Media Library, with nothing extra to license or keep current.
