Native module · Performance

SGEN tunes performance from one settings page, not a stack of plugins.

SGEN ships a native Optimization engine at one address: a vertical stack of six setting cards that control page caching, HTML, CSS, and JavaScript minification, CSS and JavaScript deferral, lazy-loading, and LCP media preloading. About fourteen controls in all, saved from a single right-rail panel.

Caching, minify, defer, lazy-load, and LCP preload are settings you toggle here, not paid plugins you install and reconcile. The six cards run in order, Caching, Minification, CSS Optimization, JavaScript Optimization, Lazy Loading, Media Preloading, and they feed a Site Vitals card on your dashboard so the tuning you do has a number you can watch.

Every control, by card

Six cards. About fourteen controls. All native.

Here is the engine card by card, with the exact control names. These are the same Core Web Vitals levers a caching plugin charges for, shipped native and saved from one page. The exclude lists and the above-the-fold skip-list are operator-entered, you tell SGEN what to leave alone.

Caching

An enable_cache switch turns page caching on.
A cache_lifespan input sets the lifespan in minutes before a fresh copy is built.

Minification

Three switches, minify_html, minify_css, minify_js.
Two exclude textareas, exclude_minify_css and exclude_minify_js, one URL per line, to skip files that should not be minified.

CSS Optimization

A defer_css switch loads stylesheets without blocking the first render.

JavaScript Optimization

A defer_js switch plus an exclude_defer_js textarea for scripts that must run early.
An inline DOMContentLoaded helper note links the external docs for the deferral pattern.

Lazy Loading

A lazyload switch defers off-screen media.
A lazyload_critical_media textarea is your above-the-fold skip-list, so hero media is not deferred.

Media Preloading

A lazyload_preload_media textarea preloads your LCP media, with inline syntax for device-prefix, fetch-priority, and crossorigin.

Questions

Performance questions, answered straight

Does SGEN replace my caching plugin? +
Yes. The Optimization engine ships page caching with a configurable lifespan, HTML, CSS, and JavaScript minification with per-URL exclude lists, CSS and JavaScript deferral, lazy-loading with an above-the-fold skip-list, and LCP media preloading, all native with nothing to install. It is the work a WordPress site usually hands to a caching plugin, here as six setting cards you save from one page.
What exactly does the Optimization engine control? +
Six cards and about fourteen controls. Caching: an enable_cache switch and a cache_lifespan in minutes. Minification: minify_html, minify_css, minify_js, plus exclude_minify_css and exclude_minify_js textareas. CSS Optimization: defer_css. JavaScript Optimization: defer_js and an exclude_defer_js textarea. Lazy Loading: a lazyload switch and a lazyload_critical_media skip-list. Media Preloading: a lazyload_preload_media textarea with device-prefix, fetch-priority, and crossorigin syntax.
Can I flush the cache from the Optimization page? +
Caching and its lifespan are configured and saved from this page. The cache is managed by the engine and the cache_lifespan setting, which rebuilds a fresh copy when the lifespan elapses, rather than by a single on-page flush button. Plan your caching through the lifespan, not a manual one-click flush.
Is the Site Vitals score live? +
No. It is an on-demand scan you run with the "Scan site vitals" button, stored as a snapshot the card labels "Last scan." It is not continuous real-time monitoring. The four tiles, First Content Paint, Avg. Load Speed, Performance Score, and Speed Index, are the last-scan read-out, the number you tune toward when you change a setting and scan again.
